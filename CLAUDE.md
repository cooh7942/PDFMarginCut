# PDFMarginCut

macOS용 PDF 여백 제거(크롭) 앱. Xcode 프로젝트.

## 기술 스택
- Swift 5.9+ / SwiftUI (앱 셸 · UI)
- PDFKit (PDF 렌더링 · 페이지 크롭)
- 최소 배포 타겟: macOS 14 (Sonoma) — `@Observable`(Observation 프레임워크) 사용을 위해 macOS 14 이상 필요
- 외부 의존성 없음(SPM 미사용). 순수 Apple 프레임워크만 사용.

## 코드 작업 워크플로우 (반드시 준수)
1. 코드를 수정하기 전에 `git checkout main` 후 `git pull`로 최신 상태로 만든다.
2. 새 브랜치를 만들어 그 위에서 작업한다. (예: `feature/overlay-view`, `fix/odd-even-margin`)
3. 코드 수정이 끝나면 **멈추고 리뷰를 기다린다.** 리뷰 전에는 commit/push하지 않는다.
4. 리뷰가 끝나면 적절한 커밋 메시지와 함께 commit & push 한다.
5. PR을 올리고 머지될 때까지 기다린다.
6. 머지되면 작업 브랜치를 삭제하고 `main`을 최신화한다.

> 브랜치 생성·리뷰 대기 없이 곧바로 main에 커밋하는 것은 금지.

## 커밋 규칙
- 커밋 메시지는 한국어 또는 영어 명령형, 한 줄 요약 + 필요 시 본문.
- 하나의 커밋은 하나의 논리적 변경만 담는다.

## 코딩 컨벤션
- 뷰/로직 분리: 크롭 계산 로직은 PDF 로직 타입(예: `PDFCropper`)에 두고 UI에서 분리한다.
- 순수 함수로 단위 테스트 가능하게 작성. 좌표 변환·크롭 계산에는 테스트를 붙인다.
- 원본 파일은 절대 직접 수정하지 않는다.
